## 命名规则

### 通用规则

1. **使用PascalCase**（大驼峰命名）
   - ✅ `HomePanel.prefab`
   - ❌ `home_panel.prefab`
   - ❌ `homepanel.prefab`

2. **文件名与内部对象名一致**
   - 预制体名称 = 文件名
   - Sprite名称 = 文件名

3. **避免使用中文**
   - ✅ `SettingsButton.prefab`
   - ❌ `设置按钮.prefab`

4. **使用有意义的名称**
   - ✅ `PlayerHealthBar.prefab`
   - ❌ `UI_001.prefab`

### UI资源命名

#### 界面预制体

**格式**：`[Module][UI Type].prefab`

**示例**：
- `HomePanel.prefab`：主界面面板
- `StartPanel.prefab`：启动界面面板
- `StoryDialog.prefab`：剧情对话框
- `LoadingScreen.prefab`：加载界面

#### UI元素

**格式**：`[Module][Element][Type].prefab`

**示例**：
- `HomePlayerInfo.prefab`：主界面玩家信息
- `HomeMapView.prefab`：主界面地图视图
- `CommonButton.prefab`：通用按钮
- `CommonDropdown.prefab`：通用下拉菜单

### 音频资源命名

#### 音效

**格式**：`SFX_[Name].wav`

**示例**：
- `SFX_Click.wav`：点击音效
- `SFX_Success.wav`：成功音效
- `SFX_Error.wav`：错误音效
- `SFX_Footstep.wav`：脚步声

#### 音乐

**格式**：`BGM_[Name].mp3`

**示例**：
- `BGM_MainTheme.mp3`：主题音乐
- `BGM_Battle.mp3`：战斗音乐
- `BGM_Town.mp3`：城镇音乐

### 纹理资源命名

#### UI纹理

**格式**：`Tex_[Module]_[Name].png`

**示例**：
- `Tex_Home_Background.png`：主界面背景
- `Tex_Button_Normal.png`：按钮常态
- `Tex_Button_Pressed.png`：按钮按下
- `Tex_Icon_Gold.png`：金币图标

#### 角色纹理

**格式**：`Tex_[Character]_[Part].png`

**示例**：
- `Tex_Player_Head.png`：玩家头像
- `Tex_Player_Body.png`：玩家身体
- `Tex_NPC_Merchant.png`：NPC商人

## Addressable 规则

### 标签（Labels）规则

#### 按模块分类

| 标签 | 用途 |
|------|------|
| `ui_start` | 启动界面资源 |
| `ui_home` | 主界面资源 |
| `ui_story` | 剧情界面资源 |
| `ui_common` | 通用UI资源 |
| `audio_sfx` | 音效资源 |
| `audio_bgm` | 背景音乐 |

#### 按加载时机分类

| 标签 | 用途 |
|------|------|
| `preload` | 预加载资源（首屏必需） |
| `ondemand` | 按需加载资源 |
| `hotupdate` | 热更新资源 |

### 分组（Groups）规则

#### 按更新频率分组

```
Groups/
├── Static/                     # 静态资源（不常更新）
│   ├── Audio
│   └── Textures
├── Dynamic/                    # 动态资源（经常更新）
│   ├── UI
│   └── Configs
└── HotUpdate/                  # 热更新资源
    ├── Scripts (game.dll.bytes)
    └── Resources
```

### Addressable Key 规则

**格式**：`[Type]/[Module]/[Name]`

**示例**：
```
UI/Home/HomePanel
UI/Common/CommonButton
Audio/SFX/Click
Audio/BGM/MainTheme
Texture/UI/Background
```

- UI预制体 → `ui_home.bundle`

```

├── ui_home.bundle
├── configs.bundle
└── textures_ui.bundle
```

## 资源引用规则

### 禁止直接引用

**❌ 不推荐**：在编辑器中拖拽引用

```csharp
public class Home : UIBase
{
    public Sprite background;  // ❌ 直接引用
}
```

### 使用 Addressable 加载

**✅ 推荐**：使用 Addressable 或 AssetManager

```csharp
public class Home : UIBase
{
    void LoadBackground()
    {
        AssetManager.Instance.Load<Sprite>("UI/Home/Background", (sprite) => {
            background.sprite = sprite;
        });
    }
}
```

## 预制体组织

### UI Prefab 规范

#### 层级结构

```
HomePanel (Canvas)
├── Background (Image)
├── PlayerInfo (Panel)
│   ├── Avatar (Image)
│   ├── Name (Text)
│   └── Level (Text)

│   └── Map (RawImage)

#### 命名规范

- **Panel**：以 "Panel" 结尾
- **Button**：以 "Button" 结尾
- **Image**：描述性名称（如 "Background"、"Avatar"）
- **Text**：描述性名称（如 "Name"、"Level"）

```
PlayerCharacter
├── Model (Mesh)
├── Animations (Animator)
├── Collider (BoxCollider)

## 最佳实践

### 1. 使用图集（Atlas）

**目的**：减少 DrawCall

```
Textures/UI/Atlas/
├── UIAtlas_Common.spriteatlas  # 通用UI图集
├── UIAtlas_Home.spriteatlas    # 主界面图集
└── UIAtlas_Battle.spriteatlas  # 战斗界面图集
```

### 2. 资源压缩

- UI纹理：使用 ASTC 压缩（移动端）
- 音效：使用 PCM 或 Vorbis
- 背景音乐：使用 Streaming 加载

### 3. 资源复用

```csharp
// ✅ 正确：复用通用组件
UI/Common/CommonButton.prefab

// ❌ 错误：重复创建
UI/Home/HomeButton.prefab
UI/Story/StoryButton.prefab
```

### 4. 及时释放资源

```csharp
void OnDestroy()
{
    // 释放加载的资源
    AssetManager.Instance.Unload("UI/Home/HomePanel");
}
```

## 总结

资源命名与组织规范定义了：

1. **命名规则**：PascalCase、有意义、避免中文
2. **目录组织**：按类型和模块分类
3. **Addressable**：标签、分组、Key规则
4. **热更新**：HotResources目录和构建规则
5. **资源引用**：使用Addressable而非直接引用

遵循这些规范可以：
- 提升资源管理效率
- 支持热更新功能
- 减少资源冲突
- 提升加载性能
