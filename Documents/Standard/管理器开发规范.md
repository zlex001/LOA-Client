# 管理器开发规范

定义LOA客户端管理器（Manager）的开发规范，包括定义、职责、初始化、事件监听等约定。

## 管理器定义

### 什么是管理器？

**管理器**（Manager）是继承 `Singleton<T>` 的核心单例类，负责管理特定领域的数据和逻辑。

**特征**：
1. 继承 `Singleton<T>`
2. 全局唯一实例
3. MonoBehaviour 生命周期
4. 持久化（DontDestroyOnLoad）

### 核心管理器

| 管理器 | 职责 | 文件位置 |
|--------|------|----------|
| Data | 数据存储和状态管理 | `Assets/Game/Scripts/Data/Data.cs` |
| Net | 网络通信和协议处理 | `Assets/Game/Scripts/Network/Net.cs` |
| UI | 界面显示和交互逻辑 | `Assets/Game/Scripts/UI/Core/UI.cs` |
| Audio | 音频播放和控制 | `Assets/Game/Scripts/Data/Audio.cs` |

## 管理器模板

### 基本结构

```csharp
using UnityEngine;
using Framework;

namespace Game
{
    public partial class [ManagerName] : Singleton<[ManagerName]>
    {
        #region Fields
        // 私有字段
        private bool isInitialized = false;
        #endregion
        
        #region Properties
        // 公共属性
        public bool IsInitialized => isInitialized;
        #endregion
        
        #region Unity Lifecycle
        void Awake()
        {
            // 注册事件监听
            RegisterListeners();
        }
        
        void Update()
        {
            // 每帧逻辑
        }
        
        void OnDestroy()
        {
            // 清理资源
            UnregisterListeners();
        }
        #endregion
        
        #region Public Methods
        public void Init()
        {
            if (isInitialized)
            {
                Debug.LogWarning($"[{GetType().Name}] Already initialized");
                return;
            }
            
            // 初始化逻辑
            
            isInitialized = true;
            Debug.Log($"[{GetType().Name}] Initialized");
        }
        #endregion
        
        #region Private Methods
        private void RegisterListeners()
        {
            // 注册事件监听
        }
        
        private void UnregisterListeners()
        {
            // 移除事件监听
        }
        #endregion
    }
}
```

## 初始化规范

### 统一初始化时机

**所有管理器在 Gate.Entrance 中统一初始化**：

```csharp
// Assets/Game/Scripts/Basic/Gate.cs
public static void Entrance(string gateway)
{
    Data.Instance.Gateway = gateway;
    
    // 初始化管理器（按依赖顺序）
    Data.Instance.Init();      // 1. 数据管理器
    UI.Instance.Init(9f, 16f); // 2. UI管理器
    Audio.Instance.Init();     // 3. 音频管理器
    Net.Instance.Init();       // 4. 网络管理器
    
    StartupFlowManager.Start();
}
```

### 初始化顺序

**原则**：被依赖的管理器先初始化

1. **Data.Instance.Init()**：数据中心，优先初始化
2. **UI.Instance.Init()**：依赖Data，第二初始化
3. **Audio.Instance.Init()**：独立管理器，第三初始化
4. **Net.Instance.Init()**：依赖Data，最后初始化

### Init 方法规范

```csharp
public void Init()
{
    // 1. 检查重复初始化
    if (isInitialized)
    {
        Debug.LogWarning($"[{GetType().Name}] Already initialized");
        return;
    }
    
    // 2. 执行初始化逻辑
    // - 设置默认值
    // - 加载配置
    // - 初始化子系统
    
    // 3. 标记已初始化
    isInitialized = true;
    
    // 4. 打印日志
    Debug.Log($"[{GetType().Name}] Initialized");
}
```

## 事件监听规范

### 在 Awake 中注册监听

**原则**：事件监听在 Awake 中注册，OnDestroy 中移除

```csharp
void Awake()
{
    // 注册 Data 监听
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
    Data.Instance.after.Register(Data.Type.LoginAccount, OnAfterLoginAccountChanged);
    
    // 注册全局事件
    Game.Event.Instance.Add(UI.Event.Click, OnUIClick);
}

void OnDestroy()
{
    // 移除 Data 监听
    Data.Instance.after.Unregister(Data.Type.Online, OnAfterOnlineChanged);
    Data.Instance.after.Unregister(Data.Type.LoginAccount, OnAfterLoginAccountChanged);
    
    // 移除全局事件
    Game.Event.Instance.Remove(UI.Event.Click, OnUIClick);
}
```

### 监听回调命名

**格式**：`On[Before/After][DataType]Changed`

```csharp
// Data 变化前
private void OnBeforOnlineChanged(params object[] args)
{
    bool online = (bool)args[0];
    // 处理逻辑...
}

// Data 变化后
private void OnAfterOnlineChanged(params object[] args)
{
    bool online = (bool)args[0];
    // 处理逻辑...
}
```

### 事件监听最佳实践

```csharp
// ✅ 正确：在 Awake 中注册
void Awake()
{
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
}

// ❌ 错误：在 Start 中注册（可能错过事件）
void Start()
{
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
}

// ❌ 错误：在 Init 中注册（Init 时机不确定）
public void Init()
{
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
}
```

## 职责划分

### 管理器职责边界

#### Data 管理器

**职责**：
- ✅ 存储游戏数据
- ✅ 提供数据访问接口
- ✅ 触发数据变化事件

**禁止**：
- ❌ 处理 UI 逻辑
- ❌ 处理网络通信（除监听后发送）
- ❌ 播放音频

#### Net 管理器

**职责**：
- ✅ 管理 Socket 连接
- ✅ 发送和接收协议
- ✅ 处理断线重连

**禁止**：
- ❌ 直接操作 UI
- ❌ 存储游戏数据（应通过 Data）
- ❌ 处理业务逻辑

#### UI 管理器

**职责**：
- ✅ 管理界面打开/关闭
- ✅ 处理 UI 交互
- ✅ 更新界面显示

**禁止**：
- ❌ 存储游戏数据（应通过 Data）
- ❌ 发送网络协议（应通过 Data 触发）
- ❌ 直接访问 Socket

#### Audio 管理器

**职责**：
- ✅ 播放音效和音乐
- ✅ 控制音量
- ✅ 管理音频资源

**禁止**：
- ❌ 处理业务逻辑
- ❌ 操作 UI
- ❌ 存储游戏数据

## 性能优化

### 避免在 Update 中频繁操作

```csharp
// ❌ 不推荐：每帧检查
void Update()
{
    if (Data.Instance.Online != lastOnlineState)
    {
        OnOnlineChanged();
        lastOnlineState = Data.Instance.Online;
    }
}

// ✅ 推荐：使用事件监听
void Awake()
{
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
}
```

### 缓存常用引用

```csharp
// ✅ 正确：缓存引用
private Data data;

void Awake()
{
    data = Data.Instance;
}

void SomeMethod()
{
    bool online = data.Online;  // 使用缓存
}

// ❌ 不推荐：频繁访问 Instance
void SomeMethod()
{
    bool online = Data.Instance.Online;  // 每次访问 Instance
}
```

### 使用对象池

```csharp
// 对于频繁创建/销毁的对象，使用对象池
public class PoolManager : Singleton<PoolManager>
{
    private Dictionary<string, Queue<GameObject>> pools = 
        new Dictionary<string, Queue<GameObject>>();
    
    public GameObject Get(string key)
    {
        if (pools.TryGetValue(key, out var pool) && pool.Count > 0)
        {
            return pool.Dequeue();
        }
        // 创建新对象...
    }
    
    public void Return(string key, GameObject obj)
    {
        if (!pools.ContainsKey(key))
            pools[key] = new Queue<GameObject>();
        
        pools[key].Enqueue(obj);
        obj.SetActive(false);
    }
}
```

## 总结

管理器开发规范定义了：

1. **管理器定义**：继承 Singleton<T> 的单例类
2. **初始化规范**：在 Gate.Entrance 中统一初始化
3. **事件监听**：在 Awake 中注册，OnDestroy 中移除
4. **职责划分**：明确各管理器的职责边界
5. **日志规范**：统一的日志格式和级别
6. **异常处理**：完善的参数验证和异常捕获
7. **性能优化**：避免频繁操作，使用缓存和对象池

遵循这些规范可以：
- 保持代码风格一致
- 提升代码可维护性
- 避免常见错误
- 提升系统性能
