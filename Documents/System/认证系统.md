# 认证系统（Authentication）

认证系统负责两组消息收发：通过 HTTP 获取服务器列表与本地化文本，通过 TCP 完成登录验证。

客户端所需的全部服务端数据：

- **服务器列表**（Servers）— 用于选服与 TCP 连接
- **本地化文本**（Texts）— 用于登录前界面的多语言显示
- **登录结果码**（Code）— 判断登录成功或失败原因
- **游客账号凭证**（AccountId + Password）— 仅快速登录成功时下发

## 二、登录请求（LoginRequest）

TCP 请求，统一覆盖账号登录与快速登录两种场景。`id` 和 `password` 为空时视为快速登录。

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | string | 账号 ID（快速登录时为空） |
| `password` | string | 密码（快速登录时为空） |
| `version` | string | 客户端版本号 |
| `platform` | string | 平台标识 |
| `device` | string | 设备标识 |
| `language` | string | 语言标识 |

---

## 三、登录响应（LoginResponse）

TCP 响应，同时作为账号登录与快速登录的统一响应。

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | int | Success / PasswordError / AppVersionUnfit / UnsafeAccount |
| `accountId` | string | 成功时下发的账号 ID |
| `password` | string | 成功时下发的密码 |
| `isGuest` | bool | 是否游客账号 |
| `isNewAccount` | bool | 是否新创建账号 |

---

## 四、待同步服务端（Server-side requirements）

以下为客户端侧已确定的接口设计，需告知服务端 / 服务端 AI Agent，尚未向服务端反馈，后续整理完再同步。

- **GatewayResponse.servers** 改为字典，不再使用结构体列表：`Dictionary<string, ServerEntry>`，key 为服务器名称的本地化 key。
- **ServerEntry** 仅保留 `ip`、`port`；不再有 `id`、`name`。
- **服务器名称** 走本地化：服务端只提供 key，客户端用该 key 在 `texts` 中取当前语言文案，以支持多语言。
- **GatewayResponse.texts** 的 key 采用**标准驼峰（camelCase）**命名（Main.cs 阶段的存量 key 使用 snake_case，保持兼容）；具体 key 列表及简体中文示例、应用界面、代码引用位置见上文「一、网关响应」中 texts 字段下方的表格（共 76 个 key，覆盖 Main 启动阶段、登录前界面、登录后固定文本），需与客户端约定一致。
