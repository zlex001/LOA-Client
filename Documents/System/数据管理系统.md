# 数据管理系统（Data）

负责客户端所有游戏数据的集中管理，是整个客户端架构的数据中心。采用数据驱动设计，其他管理器通过监听 Data 变化来响应游戏状态。

## 系统概述

数据管理系统提供：
- **游戏数据管理**：玩家信息、游戏状态的集中存储
- **配置数据加载**：策划表数据的加载与查询
- **本地数据持久化**：PlayerPrefs 存储管理
- **数据变化监听**：befor/after 事件系统

**核心文件**：
- `Assets/Game/Scripts/Data/Data.cs`：数据管理器
- `Assets/Game/Scripts/Data/Config.cs`：配置管理器
- `Assets/Game/Scripts/Data/Local.cs`：本地存储管理器

## Data（游戏数据管理器）

**Data** 是游戏数据的单例管理器，存储所有游戏状态和玩家信息。

### Data 类定义

```csharp
public partial class Data : Singleton<Data>
{
    // 数据类型枚举
    public enum Type
    {
        Ping,
        LoginAccount,
        Online,
        Home,
        SceneScale,
        Maps,
        Pos,
        Characters,
        // ...更多数据类型
    }
    
    // 数据存储
    public Dictionary<Enum, object> raw = new Dictionary<Enum, object>();
    
    // 事件系统

    public Monitor after = new Monitor();  // 变化后事件
```

#### 2. 属性访问模式

**所有数据通过属性访问，内部调用 Get/Change 方法**：

```csharp
// 属性定义
public bool Online 
{ 
    get => Get<bool>(Type.Online); 
    set => Change(Type.Online, value); 
}

public List<Server> Servers 
{ 
    get => Get<List<Server>>(Type.Servers); 
    set => Change(Type.Servers, value); 
}

public Protocol.Home Home 
{ 
    get => Get<Protocol.Home>(Type.Home); 
    set => Change(Type.Home, value); 
}
```

#### 3. Get 方法（获取数据）

```csharp
private T Get<T>(Type type)
{
    if (raw.TryGetValue(type, out var value))
    {
        return (T)value;
    }
    return default(T);
}
```

**特点**：
- 类型安全：使用泛型确保类型正确
- 空值处理：不存在时返回默认值

#### 4. Change 方法（修改数据）

```csharp
private void Change<T>(Type type, T value)
{
    // 触发变化前事件
    befor.Fire(type, value);
    
    // 修改数据
    if (raw.ContainsKey(type))
        raw[type] = value;
    else
        raw.Add(type, value);
    
    // 触发变化后事件
    after.Fire(type, value);
}
```

**特点**：
- 触发事件：修改前后触发 befor/after 事件
- 自动通知：监听者自动收到通知
- 统一入口：所有数据变化都经过此方法

## Monitor 监听机制

### befor 事件（变化前）

**befor** 事件在数据变化前触发，可用于：
- 拦截非法修改
- 记录变化前的值
- 触发预处理逻辑

**示例**：拦截非法金币值

```csharp
// 注册 befor 监听
Data.Instance.befor.Register(Data.Type.Gold, (args) => {
    int newGold = (int)args[0];
    if (newGold < 0)
    {
        Debug.LogError("[Data] Gold cannot be negative!");
        // 可以阻止修改或抛出异常
    }
});
```

### after 事件（变化后）

**after** 事件在数据变化后触发，用于：
- 响应数据变化
- 更新UI
- 发送网络协议
- 触发其他逻辑

**示例**：Net 监听 Online 状态

```csharp
// Net 管理器监听 Online 状态变化
void Awake()
{
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
}

private void OnAfterOnlineChanged(params object[] args)
{
    bool online = (bool)args[0];
    if (online)
    {
        Debug.Log("[Net] Connected to server");
        StartHeartbeat();
    }
    else
    {
        Debug.Log("[Net] Disconnected from server");
        StopHeartbeat();
    }
}
```

**示例**：UI 监听 Home 数据

```csharp
// UI 管理器监听 Home 数据变化
void Awake()
{
    Data.Instance.after.Register(Data.Type.Home, OnAfterHomeChanged);
}

private void OnAfterHomeChanged(params object[] args)
{
    var home = Data.Instance.Home;
    if (home != null)
    {
        // 更新UI显示
        UpdatePlayerInfo(home.character);
        UpdateMapInfo(home.scene);
    }
}
```

### PlayerPrefs 封装

**Local** 对 PlayerPrefs 进行封装，提供类型安全的接口：

```csharp
public void Save<T>(string key, T value)
{
    if (value is int intValue)
        PlayerPrefs.SetInt(key, intValue);
    else if (value is float floatValue)
        PlayerPrefs.SetFloat(key, floatValue);
    else if (value is string stringValue)
        PlayerPrefs.SetString(key, stringValue);
    else
        PlayerPrefs.SetString(key, JsonUtility.ToJson(value));
    
    PlayerPrefs.Save();
}

public T Load<T>(string key, T defaultValue = default)
{
    if (!PlayerPrefs.HasKey(key))
        return defaultValue;
    
    if (typeof(T) == typeof(int))
        return (T)(object)PlayerPrefs.GetInt(key);
    else if (typeof(T) == typeof(float))
        return (T)(object)PlayerPrefs.GetFloat(key);
    else if (typeof(T) == typeof(string))
        return (T)(object)PlayerPrefs.GetString(key);
    else
        return JsonUtility.FromJson<T>(PlayerPrefs.GetString(key));
}
```

## 最佳实践

### 1. 统一数据入口

**所有游戏状态统一存储在 Data.Instance**：

```csharp
// ✅ 正确：通过 Data 管理状态
Data.Instance.Online = true;
Data.Instance.Gold = 1000;

// ❌ 错误：在管理器中存储状态
public class Net : Singleton<Net>
{
    private bool isOnline;  // ❌ 不要这样做
}
```

### 2. 优先使用事件监听

**避免轮询，使用事件驱动**：

```csharp
// ❌ 不推荐：轮询检查
void Update()
{
    if (Data.Instance.Online != lastOnlineState)
    {
        OnOnlineChanged();
        lastOnlineState = Data.Instance.Online;
    }
}

// ✅ 推荐：事件监听
void Awake()
{
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
}
```

### 3. 在 Awake 中注册监听

```csharp
void Awake()
{
    // 注册监听
    Data.Instance.after.Register(Data.Type.Online, OnAfterOnlineChanged);
    Data.Instance.after.Register(Data.Type.Home, OnAfterHomeChanged);
}

void OnDestroy()
{
    // 移除监听，避免内存泄漏
    Data.Instance.after.Unregister(Data.Type.Online, OnAfterOnlineChanged);
    Data.Instance.after.Unregister(Data.Type.Home, OnAfterHomeChanged);
}
```

### 4. 类型安全

**使用泛型确保类型安全**：

```csharp
// ✅ 正确：使用属性访问
bool online = Data.Instance.Online;  // 类型安全

// ❌ 错误：直接访问字典
object value = Data.Instance.raw[Data.Type.Online];  // 需要强制转换
```

### 5. 数据验证

**在 befor 事件中进行数据验证**：

```csharp
Data.Instance.befor.Register(Data.Type.Gold, (args) => {
    int newGold = (int)args[0];
    if (newGold < 0)
    {
        Debug.LogError("[Data] Gold cannot be negative!");
        // 可以阻止修改
    }
    if (newGold > 999999999)
    {
        Debug.LogWarning("[Data] Gold exceeds maximum value!");
    }
});
```

## 总结

数据管理系统通过以下模块实现集中数据管理：

1. **Data**：游戏数据的单例管理器，提供 befor/after 事件系统
2. **Config**：配置数据管理器，加载策划表和配置
3. **Local**：本地存储管理器，封装 PlayerPrefs

**核心设计理念**：
- **数据驱动**：Data 是唯一数据源
- **事件驱动**：通过 Monitor 事件系统通知变化
- **统一入口**：所有数据通过 Data.Instance 访问

这种设计确保了数据的一致性和可维护性，支撑整个客户端架构的响应式特性。
