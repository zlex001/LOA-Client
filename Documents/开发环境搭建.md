# 开发环境搭建

本文档详细说明如何搭建 LOA 客户端开发环境。

## 环境要求

### 必需软件

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| **Unity** | 2021.3 LTS 或更高 | 游戏引擎 |
| **Visual Studio** | 2022 或更高 | C# IDE（Windows推荐） |
| **JetBrains Rider** | 最新版 | C# IDE（跨平台推荐） |
| **SVN** | 1.14+ | 版本管理 |
| **.NET SDK** | 6.0+ | C# 编译 |

### 可选软件

| 软件 | 用途 |
|------|------|
| Unity Hub | Unity 版本管理 |
| Android SDK | Android 构建 |
| Xcode | iOS 构建（仅macOS） |

### 系统要求

#### Windows

- Windows 10/11（64位）
- 内存：16GB 以上
- 磁盘：50GB 可用空间
- 显卡：支持 DirectX 11

#### macOS

- macOS 12 Monterey 或更高
- 内存：16GB 以上
- 磁盘：50GB 可用空间
- Apple Silicon 或 Intel 芯片

#### Linux

- Ubuntu 20.04+ / Fedora 35+
- 内存：16GB 以上
- 磁盘：50GB 可用空间

## 安装 Unity

### 方法一：通过 Unity Hub（推荐）

1. 下载 Unity Hub：https://unity.com/download
2. 安装 Unity Hub
3. 在 Unity Hub 中安装 Unity 2021.3 LTS
4. 安装以下模块：
   - Android Build Support（Android 构建）
   - iOS Build Support（iOS 构建，仅 macOS）
   - WebGL Build Support（可选）

### 方法二：直接安装

1. 下载 Unity 2021.3 LTS：https://unity.com/releases/editor/archive
2. 运行安装程序
3. 选择安装模块（同上）

## 打开项目

### 通过 Unity Hub

1. 启动 Unity Hub
2. 点击左侧的"项目"
3. 点击右上角的"添加"按钮
4. 选择 `LOA-Client/trunk` 目录
5. 点击项目名称打开

### 直接打开

```bash
# Windows
"C:\Program Files\Unity\Hub\Editor\2021.3.x\Editor\Unity.exe" -projectPath "LOA-Client/trunk"

# macOS
/Applications/Unity/Hub/Editor/2021.3.x/Unity.app/Contents/MacOS/Unity -projectPath "LOA-Client/trunk"

# Linux
~/Unity/Hub/Editor/2021.3.x/Editor/Unity -projectPath "LOA-Client/trunk"
```

### 首次打开

首次打开项目时，Unity 会：
1. 导入所有资源（约 5-10 分钟）
2. 生成 Library 目录
3. 编译脚本

**注意**：首次导入耗时较长，请耐心等待。

## 编辑器配置

### HybridCLR 插件安装

1. 打开 Unity 项目
2. 菜单：`Window → Package Manager`
3. 点击左上角 "+" 按钮
4. 选择 "Add package from git URL"
5. 输入：`https://github.com/focus-creative-games/hybridclr_unity.git`
6. 点击 "Add"

### HybridCLR 初始化

1. 菜单：`HybridCLR → Installer`
2. 点击 "Install"
3. 等待安装完成
4. 重启 Unity

### 构建设置

1. 菜单：`File → Build Settings`
2. 选择目标平台（Android/iOS/Windows）
3. 点击 "Switch Platform"
4. 添加场景：
   - `Assets/Framework/Main.unity`（启动场景）

### Player 设置

1. 菜单：`Edit → Project Settings → Player`
2. 配置以下项：
   - **Company Name**：公司名称
   - **Product Name**：LOA
   - **Bundle Identifier**：com.company.loa
   - **Version**：1.0.0

### Scripting Define Symbols

1. 菜单：`Edit → Project Settings → Player → Other Settings`
2. 在 `Scripting Define Symbols` 中添加：
   ```
   UNITY_2021;HYBRIDCLR
   ```

## 配置本地服务器

### 方法一：修改 Config.asset

1. 打开 `Assets/Framework/Config.asset`
2. 修改 `Gateway` 字段：
   ```
   Gateway: 127.0.0.1
   或
   Gateway: 192.168.1.100（局域网服务器IP）
   ```
3. 保存文件

### 方法二：在代码中修改

编辑 `Assets/Framework/Main.cs`：

```csharp
void Start()
{
    // 覆盖配置文件中的网关地址
    config.Gateway = "127.0.0.1";  // 本地服务器
    
    // 继续原有逻辑...
}
```

## 首次运行

### 运行游戏

1. 在 Unity 编辑器中点击 Play 按钮
2. 等待以下过程：
   - 语言检测
   - 设备认证
   - 版本检查
   - 资源加载
3. 看到启动界面表示成功

### 跳过热更新（开发模式）

在编辑器中运行时，系统会自动跳过热更新，直接加载本地编译的代码。

### 常见启动问题

#### 问题1：连接认证服务器失败

**原因**：
- 服务器未启动
- 网关地址配置错误
- 防火墙阻止连接

**解决**：
1. 检查服务器是否运行：`http://[Gateway]:8880/api/health`
2. 检查防火墙设置
3. 确认网关地址正确

#### 问题2：版本检查失败

**原因**：
- 服务器未部署热更新资源
- 版本接口异常

**解决**：
1. 检查版本接口：`http://[Gateway]:8880/api/hot/version`
2. 确认服务器已部署热更新资源

#### 问题3：程序集加载失败

**原因**：
- HybridCLR 未正确安装
- game.dll.bytes 不存在或损坏

**解决**：
1. 重新安装 HybridCLR：`HybridCLR → Installer`
2. 重新构建热更新 DLL：`HybridCLR → Build → BuildHotUpdate`
3. 检查 `Assets/StreamingAssets/game.dll.bytes` 是否存在

## IDE 配置

### Visual Studio 配置

1. 安装 Unity Tools for Visual Studio
2. 在 Unity 中设置外部编辑器：
   - `Edit → Preferences → External Tools`
   - `External Script Editor` 选择 Visual Studio

3. 生成项目文件：
   - `Edit → Preferences → External Tools`
   - 勾选 "Generate .csproj files for"
   - 点击 "Regenerate project files"

### JetBrains Rider 配置

1. 安装 Rider
2. 在 Unity 中设置外部编辑器：
   - `Edit → Preferences → External Tools`
   - `External Script Editor` 选择 Rider

3. Rider 会自动识别 Unity 项目

### VSCode 配置

1. 安装 VSCode
2. 安装扩展：
   - C# (Microsoft)
   - Unity Code Snippets
   - Debugger for Unity

3. 在 Unity 中设置外部编辑器：
   - `Edit → Preferences → External Tools`
   - `External Script Editor` 选择 VSCode

## 调试配置

### Unity 编辑器调试

1. 在 IDE 中设置断点
2. 菜单：`Debug → Attach to Unity`
3. 选择 Unity 进程
4. 在 Unity 中点击 Play
5. 触发断点

### 真机调试

#### Android 调试

1. 启用 USB 调试
2. 连接设备
3. 在 Unity 中：`File → Build Settings → Build and Run`
4. 使用 Android Studio 的 Logcat 查看日志

#### iOS 调试

1. 连接 iOS 设备
2. 在 Unity 中：`File → Build Settings → Build`
3. 使用 Xcode 打开生成的项目
4. 运行并调试

## 团队协作

### SVN 工作流

1. 每次开发前先 Update：
   ```bash
   svn update
   ```

2. 完成开发后提交：
   ```bash
   svn commit -m "[功能] 添加XXX功能"
   ```

3. 冲突处理：
   - 手动解决冲突
   - 使用 Unity Smart Merge 工具

### .meta 文件

- ✅ 必须提交 `.meta` 文件
- ❌ 不要删除 `.meta` 文件
- ⚠️ `.meta` 文件冲突需谨慎处理
