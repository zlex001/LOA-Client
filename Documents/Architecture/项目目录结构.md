## 二、AOT 区（编译进主包）

AOT 区的内容编译进安装包，更新需重新提交应用商店。

### 1. Launcher

**路径：** `Assets/Launcher/`

应用启动入口，无 `.asmdef`，编译进默认 Assembly-CSharp。

| 文件 | 职责 |
|------|------|
| `Main.unity` | Unity 主场景，挂载 Main 组件 |
| `Main.cs` | 入口 MonoBehaviour：设备认证 → 热更新检查（编辑器跳过）→ 加载程序集 → 启动 Gate |
| `Config.cs` + `Config.asset` | ScriptableObject：网关地址、版本号、环境切换 |

### 2. Framework

**路径：** `Assets/Framework/`
**程序集：** `Framework.asmdef`（AOT）

稳定基础设施，极少变更。

| 文件 | 职责 |
|------|------|
| `Singleton.cs` | 泛型单例基类 |
| `AssetManager.cs` | 资源加载管理 |
| `AssetManagerExtension.cs` | AssetManager 编辑器扩展 |
| `AssetDependData.cs` | AssetBundle 依赖数据结构 |
| `Http.cs` | HTTP 请求 |
| `Hot.cs` | 热更新引擎 |
| `Path.cs` | 路径常量 |
| `Loom.cs` | 线程调度 |
| `Localization.cs` | 多语言管理（读取 Resources/ 下的 JSON） |
| `LanguageDetector.cs` | 系统语言检测 |
| `File.cs` | 文件工具（MD5 等） |
| `DirectoryUtils.cs` | 目录操作工具 |
| `FontScaler.cs` | 字体缩放 |
| `ZipSharp.cs` | ZIP 解压 |

### 3. Resources

**路径：** `Assets/Resources/`

Unity 特殊目录，通过 `Resources.Load()` 加载。仅存放登录前需要的静态数据。

内容：`Localization_{Language}.json`（23 种语言）、IAP 配置。

### 4. Plugins

**路径：** `Assets/Plugins/`

原生平台插件和预编译 DLL。

| 子目录 | 内容 |
|--------|------|
| `Android/` | Android 平台原生插件 |
| `IngameDebugConsole/` | 运行时调试控制台 |
| `libs/` | 预编译 DLL |

### 5. StreamingAssets

**路径：** `Assets/StreamingAssets/`

AOT 补充元数据 DLL（`Framework.dll.bytes` 等），用于 HybridCLR 运行时元数据注册。

### 6. TextMesh Pro

**路径：** `Assets/TextMesh Pro/`

Unity TextMeshPro 包的默认资源，含字体、Shader、示例。不修改。

## 三、热更新区（运行时可更新）

热更新区的内容可通过热更新机制单独下载更新，无需重新提交应用商店。

### 1. Game/Scripts

**路径：** `Assets/Game/Scripts/`

8 个热更新程序集 + 第三方库。详细分层架构见 `Documents/客户端架构文档.md`。

```
Scripts/
├── Utils/              工具层
├── Basic/              基础设施层（Event、Monitor、Flow）
├── Logic/              业务数据层（Data、Config、Models）
├── Network/            网络层（Net、Protocol、ProtocolConverter）
├── Domain/             领域层
├── UI/                 表现层
├── Start/              启动层（Gate）
├── SDK/                平台 SDK
├── ThirdParty/         第三方库（LitJson、DOTween、SuperScrollView）
└── HybridCLRGenerate/  HybridCLR 自动生成（AOTGenericReferences、link.xml）
```

### 2. Game/HotResources

**路径：** `Assets/Game/HotResources/`

游戏资源的唯一存放目录。构建时由 `Build.SetBundleName()` 扫描并打包为 AssetBundle，运行时由 `AssetManager` 从热更新路径加载。

| 子目录 | 内容 |
|--------|------|
| `Audio/` | BGM、SFX 音频 |
| `Prefabs/` | UI Prefab、特效 Prefab |
| `RawAssets/` | 贴图、材质等原始资源 |

## 五、新增文件归属规则

新增文件或文件夹时，按以下规则判断归属：

| 问题 | 归属 |
|------|------|
| 是否为热更新游戏代码？ | `Game/Scripts/{对应层}/` |
| 是否为可热更新的资源（Prefab/音频/贴图）？ | `Game/HotResources/` |
| 是否为编辑器工具？ | `Editor/` |
| 是否为登录前需要的静态数据？ | `Resources/` |
| 是否为原生平台插件？ | `Plugins/` |
| 是否为启动流程代码？ | `Launcher/`（极少修改） |
| 是否为框架基础设施？ | `Framework/`（极少修改） |

**禁止：**
- 在 `Assets/` 根目录下新建顶层文件夹（除非有架构级别的理由）
- 将游戏代码放在 `Game/Scripts/` 以外的位置
- 将运行时资源放在 `Editor/` 下
