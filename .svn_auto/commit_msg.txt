## 核心架构变更（已完成）

### 1. 数据分层重构  
- Protocol从独立层移至Net层（Assets/Game/Scripts/Network/Protocol/）
- Data层重命名为Logic层，创建独立数据模型
- 新增9个Logic数据模型类：MapData, SceneData, HomeData, CharacterData, InitializeData, OptionData, WorldMapData, UILockData, TutorialData, DialogueData, InformationData
- 所有数据模型字段统一使用camelCase（pos, name, scene等）

### 2. 类型系统调整
- 创建ProtocolConverter转换器（Protocol → Logic）
- 重构所有Protocol.Processed()方法使用转换器
- OptionItem数据模型改名为OptionItemData（避免与UI组件OptionItem冲突）
- UI层全面从Protocol类型迁移到Logic类型

### 3. 程序集依赖配置
**新增程序集：**
- LitJson.asmdef（源代码程序集，位于ThirdParty/Json/）

**依赖关系更新：**

- Net.asmdef: Protocol, Logic, Basic, Framework, Utils + Google.Protobuf.dll
- Presentation.asmdef: 新增LitJson, Google.Protobuf.dll, UnityEngine.Purchasing
- 移除所有非Net层对Protocol的直接引用

### 4. UI层适配（50+文件）
**类型转换：**

**组件修复：**

## 已知问题及后续工作

### 待解决：DOTween扩展方法识别问题（约25个编译警告）
**现象：**
```
error CS1929: 'Image' does not contain a definition for 'DOFade'
error CS1061: 'RectTransform' does not contain a definition for 'DOAnchorPosX'
```

**影响文件：**
- MapTransition.cs, Option.cs, Start.cs, StartSettings.cs, UI.cs

**原因分析：**
- DOTween.dll已正确配置在Presentation.asmdef的precompiledReferences中
- 所有文件已包含`using DG.Tweening;`
- DLL本身包含这些方法（已验证）
- Unity编译器未识别扩展方法（可能是程序集加载时机问题）

**解决方案：**
1. Unity退出安全模式后执行：Assets → Reimport All
2. 如仍有问题，可能需要DOTween完整安装包（包含ModuleUI等）
3. 或检查DOTween是否需要在PlayerSettings中配置
